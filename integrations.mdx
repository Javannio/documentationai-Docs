## GitHub Version Control

Connect your documentation to GitHub for collaborative editing and automatic publishing.

<Tabs>
  <Tab title="Connect Repository" icon="link">
    1. Navigate to Settings > Integrations.
    2. Authorize GitHub and select your repo.

    <CodeGroup tabs="CLI,Web">
    ```bash
    git remote add docs https://github.com/your-org/zorisord-docs.git
    git push docs main
    ```
    ```javascript
    // Webhook trigger on push
    await fetch('https://api.example.com/webhooks/github', {
      method: 'POST',
      headers: { 'Authorization': 'Bearer YOUR_TOKEN' }
    });
    ```
    </CodeGroup>
  </Tab>
  <Tab title="Sync Changes" icon="git-branch">
    Pull requests automatically trigger doc builds.
  </Tab>
</Tabs>

## Webhook Setup

Set up webhooks to automate notifications for infrastructure project updates.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Go to Settings > Webhooks > Add New.
    
    Set the payload URL to your endpoint, like `https://your-webhook-url.com/zorisord`.
  </Step>
  <Step title="Configure Events" icon="settings">
    Select events: Page Update, Deployment, User Invite.
  </Step>
  <Step title="Test Delivery" icon="zap">
    Trigger a test event and verify receipt.
  </Step>
</Steps>

<Request tabs="Page Update,Deployment" show-lines="true">
```javascript
POST /webhooks/zorisord HTTP/1.1
Host: your-webhook-url.com
Content-Type: application/json
X-Signature: hmac-sha256=YOUR_SIGNATURE

{
  "event": "page_update",
  "space": "zorisord-dev-team",
  "path": "/integrations.mdx",
  "timestamp": "2024-10-15T10:00:00Z"
}
```
```javascript
POST /webhooks/zorisord HTTP/1.1
Host: your-webhook-url.com
Content-Type: application/json

{
  "event": "deployment",
  "version": "v1.2.0",
  "status": "success",
  "url": "https://docs.example.com"
}
```
</Request>

<Response tabs="200,400">
```json
{
  "status": "received",
  "event": "page_update",
  "id": "webhook_123"
}
```
```json
{
  "error": "Invalid signature",
  "code": 400
}
```
</Response>

## API Access for Custom Extensions

Use our API to fetch documentation or trigger builds programmatically.

<ParamField path="space" param-type="string" required="true">
  Your documentation space identifier, e.g., `zorisord-dev-team`.
</ParamField>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Bearer YOUR_API_KEY`.
</ParamField>

<CodeGroup tabs="JavaScript,Python">
```javascript
const response = await fetch('https://api.example.com/v1/spaces/zorisord-dev-team/pages', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});
const pages = await response.json();
console.log(pages);
```
```python
import requests

headers = {'Authorization': 'Bearer YOUR_API_KEY'}
response = requests.get('https://api.example.com/v1/spaces/zorisord-dev-team/pages', headers=headers)
pages = response.json()
print(pages)
```
</CodeGroup>

<Expandable title="Advanced API Filtering" default-open="false">
  Add query params like `?updated_after=2024-10-01` for recent changes.

  ```javascript
  fetch('https://api.example.com/v1/pages?updated_after=2024-10-01&limit=50')
  ```
</Expandable>
